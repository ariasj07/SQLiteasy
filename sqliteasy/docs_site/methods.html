create_database()

The create_database() method is the first step you want to do, after creating a new SQLiteasy class intance. It establishes a connection to the database specified in the class constructor, creates a cursor for executing SQL commands, executes the table creation process, and finally closes the connection to ensure proper resource management.

...
database.create_database()
...

fetch_database(indent=bool)

The fetch_database() method prints and return all the data stored in the database by executing a SELECT * FROM database.db query. This method serves two purposes: it prints the data to the console and also returns it as a result. The indent parameter, which is a boolean, determines the format of the printed output. When indent=True, the data is displayed in a well-formatted and readable manner. If indent=False, the data is printed in a compact, unformatted style. This method is particularly useful for quickly inspecting or processing the entire dataset.

all_users = fetch_database(indent=True)

fetch_database_by()

The fetch_database_by(where, by, indent=bool) method allows you to retrieve a specific row from the database based on a condition. You specify the column name and the value to match. For example:

user_josue = fetch_database_by("user_name", "josue", indent=True)

This will return and print the row where the value in the user_name column is "josue". The indent parameter determines the format of the printed output. When indent=True, the data is displayed in a well-formatted and readable manner. If indent=False, the data is printed in a compact, unformatted style. This method is useful for retrieving and inspecting specific rows of data efficiently.

fetch_database_by_and()

This method prints and return the row where both specified conditions are True. It uses an AND logic, meaning that for the row to be returned, both conditions must be satisfied simultaneously. If condition 1 is True and condition 2 is also True, the method will return the matching row. However, if either condition 1 or condition 2 is False, the method will not return any results. This ensures pre class="alert alert-light"cise filtering based on multiple criteria.

users_with_pro_plan_and_registered = database.fetch_database_by_and("user_isregistered", "1", "user_has_pro_plan", "1", indent=True)

fetch_database_by_or()

The fetch_database_by_or() method prints and return rows where at least one of the specified conditions is True. It operates using OR logic, meaning that if either condition 1 or condition 2 is satisfied, or if both are satisfied, the method will return the matching rows. This makes it useful for scenarios where you need to filter data based on multiple optional criteria. For example, if condition 1 matches a specific value in one column and condition 2 matches a value in another column, the method will include rows that meet either or both conditions in the result.

users_with_pro_plan_or_registered = database.fetch_database_by_or("user_isregistered", "1", "user_has_pro_plan", "1", indent=True)

insert_database()

The insert_database() method enables you to add new data entries into the database. It is essential to specify the parameter names and provide values for all the columns defined in the database. For instance, if your database contains columns named "user_name" and "user_email", you must include both parameters when calling this method. This ensures that the data is inserted correctly and aligns with the database schema.

database.insert_database("user_name"="New user", "user_email"="newuseremail@gmail.com")

update_database()

The update_database() method allows you to modify a specific value in the database. It requires the following parameters:

  • column_to_update: The column whose value will be updated.
  • condition_column: The column used to identify the row(s) to update.
  • condition_value: The value in the condition_column that must match for the update to occur.
  • new_value: The new value to assign to the specified column.

This method is useful for making targeted updates to your database while ensuring that only the intended rows are modified.

database.update_database(column_to_update="user_has_pro_plan", condition_column="user_name", condition_value="josue", new_value="1")
# I'm giving the pro plan to the user "josue"
